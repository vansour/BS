\chapter{实验设计与结果分析}

\section{实验目的}

本文实验并非仅为了给出单一数值，而是围绕以下四个目标展开：

\begin{enumerate}[leftmargin=2.5em]
\item 验证“深度缓存 + 在线造雾 + 统一多任务模型”主链路能够正常运行；
\item 验证统一模型是否能够同时输出车辆检测、雾类型分类和 beta 回归结果；
\item 分析在线造雾与多任务学习对于固定监控视频团雾监测的适配性；
\item 在真实下载视频上完成系统级演示验证，并评估工程可用性与后续改进方向。
\end{enumerate}

与标准竞赛型论文不同，本文实验更加注重“方法合理性 + 工程闭环 + 可视化验证 + 后续可扩展性”。因此，实验评价体系也分为定量评价与系统验证两部分。

当前项目运行环境为：

\begin{itemize}[leftmargin=2em]
\item Python 3.13.12
\item PyTorch 2.12.0.dev20260406 + CUDA 13.0
\item torchvision 0.27.0.dev20260406
\item Ultralytics 8.4.34
\item NumPy 2.4.3
\item GPU：NVIDIA GeForce RTX 5070 Ti Laptop GPU，显存约 12 GB
\end{itemize}

在配置方面，本文当前统一模型采用的关键参数包括：

\begin{itemize}[leftmargin=2em]
\item 基础检测权重：\texttt{yolo11n.pt}
\item 检测类别数：1（vehicle）
\item 雾分类类别数：3（clear / uniform / patchy）
\item 输入尺寸：512
\item beta 范围：0.02 \textasciitilde{} 0.10
\item 基础置信度阈值：0.25
\item EMA 平滑系数：0.1
\end{itemize}

从工程视角看，这样的配置具有一定代表性：一方面能够在 12GB 级别显存下完成训练与推理；另一方面也为后续迁移到更轻量设备提供了参考起点。

\section{评价指标设计}

为了较全面评价本文方法，实验指标设计分为三部分。

第一，检测任务指标。后续完整实验中应使用 Precision、Recall、mAP@0.5 和 mAP@0.5:0.95 作为标准检测指标，同时结合雾天视频上的实际框稳定性进行定性分析。由于当前统一模型仍在进一步续训，本文初稿阶段重点保留检测评估框架，并对已有早期权重在真实雾视频上的表现进行工程分析。

第二，雾分类任务指标。对于 \texttt{clear / uniform / patchy} 三分类任务，可采用 Accuracy、Macro-F1、混淆矩阵等指标。其中，\texttt{patchy} 类别的识别能力尤为重要，因为它更贴近高速公路团雾的实际风险特征。

第三，beta 回归任务指标。由于 beta 为连续变量，适合采用 MSE、MAE、RMSE 以及与类别等级的一致性分析作为评价方法。考虑到当前 beta 主要来自在线合成监督，因此其数值意义更侧重“相对强弱”而非绝对气象站量测值，后续若能引入真实能见度标定数据，则可进一步采用与真实能见度或散射参数的相关性进行校验。

除上述定量指标外，本文还特别重视系统级验证指标，包括：推理速度、视频导出稳定性、复杂场景可视化质量、混合推理的工程可用性、右下角雾浓度分布图的解释效果等。

\section{训练流程与当前状态分析}

在训练组织上，本文使用 \texttt{src/train.py} 作为统一入口，并实现了以下训练特性：

\begin{itemize}[leftmargin=2em]
\item 支持 checkpoint 自动恢复；
\item 支持训练和验证缺失深度缓存自动补算；
\item 支持 AMP 混合精度训练；
\item 支持对非有限梯度进行监控与恢复；
\item 支持保存 \texttt{unified\_model.pt} 与 \texttt{unified\_model\_best.pt}；
\item 支持后续进入 QAT 和 INT8 转换流程。
\end{itemize}

当前工作区中最早的 checkpoint 记录显示，模型在首个 epoch 完成后保存，训练损失约为 6.9528，验证损失约为 6.3626。这个结果说明统一模型主链已经具备收敛基础，但尚处于早期阶段。后续实际推理表明，该阶段权重在雾分类和 beta 回归方面可以输出稳定数值，但在真实雾视频上的车辆检测性能仍不够理想，因此本文在系统演示阶段一度采用混合推理完成车辆与雾状态的联合可视化。

在本稿写作过程中，项目进一步完成了一轮带边界的快速微调。该微调仍从现有 checkpoint 继续训练，但将单个 epoch 的训练 batch 限制为 600、验证 batch 限制为 150，以便在较短时间内获得更新权重。训练结果显示，第二个可用 epoch 的平均训练损失已下降到 3.1791，验证损失为 7.8716；继续训练后，在第三个 epoch 上训练损失进一步下降到 2.5315，验证损失下降到 2.5609，并刷新当前最佳模型。若进一步观察损失分解项，可见检测损失虽仍占主导，但雾分类损失已从早期大约 1.1 的水平继续降低到 0.344 左右，beta 回归损失也维持在较小量级。这说明本文提出的多任务组织方式不仅可训练，而且在短程续训后已经明显改善了雾任务的收敛状态。

根据这次快速微调生成的 \texttt{summary.json} 和 \texttt{metrics.jsonl}，系统目前已完成 FP32 阶段的受限续训，并得到新的 \texttt{unified\_model\_best.pt} 与 \texttt{checkpoint\_epoch\_0003.pt}。因此，从论文撰写角度看，本文已不再只是“方法设计 + 代码打通”，而是已经拥有一版经过继续优化的可用权重，为后续补充更完整的定量实验结果提供了基础。

\section{当前视频推理结果分析}

为了评估系统在真实下载视频上的适用性，本文选取根目录中的雾天交通视频片段进行推理验证。该视频时长约 28 秒，分辨率为 768×432，帧率为 30 FPS，内容为固定机位下的浓雾道路监控场景，符合“固定监控视角 + 雾天气 + 车流经过”的研究需求。

首先，直接使用当前统一模型权重在抽样帧上进行推理，系统能够输出相对稳定的雾分类概率和 beta 数值。对若干代表帧的抽样结果显示，beta 大约稳定在 0.050 左右，雾类别概率集中在 \texttt{uniform} 与 \texttt{patchy} 附近，说明模型已经感知到该视频并非无雾场景。尽管数值本身还不能直接替代真实能见度量测，但它为后续通过 EMA 平滑和动态阈值进行联动提供了基础。

其次，直接让当前统一模型承担车辆检测任务时，在该真实雾视频上的检测结果偏弱，部分抽样帧检测数接近 0。这与模型当前训练轮次较少、合成雾到真实雾仍存在域偏移有关。为验证系统其他部分的工程可行性，本文进一步用原始预训练 \code{yolo11n.pt} 在相同视频上进行车辆检测测试。结果表明，预训练 YOLO11n 在中后段车流密集区段可稳定检出多辆车、卡车和公交车，说明真实视频场景中的车辆目标本身是可检测的，问题主要集中在统一模型的联合训练尚需进一步加强。

再次，本文实现的混合推理策略能够较好解决阶段性效果不均衡问题。该策略使用 \code{yolo11n.pt} 负责车辆检测，统一模型负责雾分类和 beta 估计，并将二者结果同时绘制到视频中。从系统效果上看，输出视频已经能够在同一画面中展示车辆检测框、雾状态文字、beta 数值以及右下角雾浓度分布图，形成了较强的演示完整性和可解释性。

\section{雾浓度分布图的小地图分析}

在本文后续工程实现中，系统进一步将右下角的简单 beta 数字扩展为带中文标题、颜色刻度条和分级标签的“雾浓度分布图”。该面板包含：

\begin{itemize}[leftmargin=2em]
\item 中文标题“雾浓度分布图”；
\item 低、中、高三段颜色刻度条；
\item 基于均值分档的“轻雾 / 中雾 / 浓雾”标签；
\item 全局 beta 数值与局部相对雾强度小地图。
\end{itemize}

需要再次说明的是，这张小地图并非模型直接预测的逐像素浓度图，而是利用当前帧图像的局部对比度、饱和度和亮度白化程度构建的相对空间雾浓度估计，并使用全局 beta 做强度校正。从科学严谨性上说，它还不能替代真实空间能见度反演；但从系统演示和交互解释上看，它非常适合体现“同一画面不同位置雾浓度不同”的业务直觉。对于毕业设计答辩而言，这种设计可显著提高系统结果的可读性。

\section{方法优势分析}

结合当前工程实现与初步结果，本文方法具有如下优势。

第一，数据利用效率高。通过复用清晰交通监控数据和在线造雾机制，系统避免了对大规模真实团雾标注数据的强依赖，降低了研究门槛。

第二，任务组织紧密。统一模型将检测、分类和回归整合到同一特征空间中，使天气感知与目标感知不再割裂。

第三，工程链路完整。当前项目不仅能训练，还具备数据自检、深度缓存补算、视频推理、混合推理、导出和量化路径，说明它不是纸面模型，而是可运行系统。

第四，具有较强扩展性。后续既可以继续强化统一模型训练，也可以引入真实雾视频做微调，甚至将右下角小地图替换为真正的密集雾浓度预测头。

\section{当前不足与原因分析}

尽管本文已经形成较完整的研究和工程框架，但当前系统仍存在以下不足。

第一，统一模型对真实雾视频中的车辆检测能力仍然偏弱。根本原因在于当前训练轮次较少、真实雾域与合成雾域仍有差距，而且多任务联合训练中检测损失占主导，天气头与检测头之间的最优平衡尚未完全建立。

第二，beta 回归的物理标定尚不充分。本文中的 beta 来自合成监督，虽然有连续性与可解释性，但尚未与真实气象站能见度、透射率或 LIDAR 观测建立严格映射关系。

第三，右下角空间雾浓度小地图目前是相对估计而非端到端预测。它更适合作为工程解释增强，而不能直接作为严谨的科学测量结果。

第四，实验对比仍需在完整训练结束后进一步完善。包括不同 backbone 的对比、是否使用深度缓存的消融、是否使用 patchy 模式的消融、不同损失权重的影响、动态阈值机制的作用等，都值得在终稿阶段补充更完整的定量表格。
