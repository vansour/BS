\chapter{结论与展望}

本文围绕高速公路团雾监测这一典型低能见度智能交通问题，提出并实现了一套基于深度估计与多任务学习的统一监测方法。针对真实团雾样本稀缺、浓度标签难以获取、固定监控视频场景与自动驾驶数据差异明显等问题，本文选择以 UA-DETRAC 清晰交通数据为基础，通过 \code{MiDaS_small} 预计算深度缓存，构建基于大气散射模型的在线造雾机制，在训练阶段动态生成 \code{clear}、\code{uniform}、\code{patchy} 三类天气样本，并进一步设计统一多任务模型同时完成车辆检测、雾类型分类和 beta 浓度回归。与单一任务研究相比，本文更强调“统一建模 + 视频级解释 + 工程闭环”三者的结合\cite{uadetrac,midas,koschmieder,caruana1997multitask}。

结合当前代码实现与工作区产物，本文已形成以下几项较为明确的结论。

\begin{enumerate}[leftmargin=2.5em]
\item \textbf{统一多任务主链已经稳定打通。} 当前系统能够稳定读取原始图像、XML 标注与深度缓存，执行在线造雾、多任务训练、checkpoint 保存与续训，并在推理侧完成 letterbox 预处理、三任务联合前向、NMS、时序过滤、beta 平滑和动态阈值控制。
\item \textbf{雾分支已经从“能跑”提升到“可作为正式论文结果引用”。} \code{fogfocus_full} 正式训练将最佳验证损失稳定优化到 \code{1.0072}，且未出现 non-finite gradient 或 AMP recovery，说明冻结检测器后，雾分类与 beta 回归分支能够在完整 epoch 条件下继续稳定收敛。
\item \textbf{统一模型在雾感知上已有明显进展，但检测分支仍是主要短板。} 真实雾视频和 \code{benchmark_v1} 的正式结果均表明，\code{fogfocus} 与 \code{fogfocus_full} 已能打破默认模型“全判 clear”的塌缩问题，并稳定输出非 \code{clear} 主导的雾类别与连续 beta；然而在车辆检测层面，统一路线仍明显弱于独立 \code{yolo11n.pt} 支撑的混合路线。与此同时，新增的 UA-DETRAC 标准检测表又说明：默认 unified 模型在同域晴天验证集上其实拥有最高的 Precision、Recall 和 mAP。这进一步证明，当前问题并不是“统一模型完全不会检测车”，而是“如何把清晰天气域上的检测能力与真实雾视频中的完整路线稳定性统一起来”。
\item \textbf{混合推理是当前阶段最务实的最终工程路线。} 截至当前正式基准评估，\code{default_baseline}、\code{fogfocus}、\code{videoadapt} 和 \code{fogfocus_full} 在 5 段 \code{benchmark_v1} 视频上的推荐路线全部仍为 \code{hybrid}。这说明当前最佳工程方案不是强行要求统一模型同时把雾感知与检测任务都做到最优，而是由统一模型承担更稳定的雾识别与 beta 输出，再由成熟独立检测器兜底车辆检测。
\item \textbf{静止假车误检治理已经从启发式想法发展为闭环工程链。} 当前项目已经落地了轨迹级时序过滤、可疑静止轨迹 patch 导出、人工复核包构建、bootstrap hard negative 收口以及二次车辆 / 非车辆复核分类器训练，并且相关权重已经能够回接到 \code{TemporalVehicleFilter} 中参与真实推理链。
\item \textbf{失败分支同样具有研究价值。} \code{clearpreserve_pilot}、\code{detectorrecovery_pilot} 和 \code{cocohead_pilot} 三条检测能力回拉路线全部未通过正式候选模型基准评估的门槛判定，这一结果反而帮助本文明确了统一模型当前问题并非简单超参数微调即可修复，而更可能与真实雾域和合成雾域差距、检测头长期冻结后的表示退化以及晴天对照视频片段上的检测能力缺口有关。
\end{enumerate}

从研究贡献角度看，本文最重要的价值并不只是给出一版权重文件，而是给出了一种较为完整的问题处理框架。该框架在数据层面提供了“晴天监控视频复用 + 深度缓存 + 在线造雾”的监督构造策略，在模型层面提供了“检测 + 雾分类 + beta 回归”的统一多任务建模方式，在工程层面则提供了从训练、推理、基准评估到误检治理与回灌的全链路原型。对于毕业设计而言，这种“方法、代码、实验与应用场景相互闭合”的状态，比单独展示少量可视化截图更有说服力。

当然，本文仍处于原型系统阶段，后续工作至少还可以从以下几个方向继续推进。

首先，应继续扩充真实雾域验证集。目前 \code{benchmark_v1} 已经从单视频模板扩展到 5 段有效视频资产，但其中真实雾视频仍只有 1 段。下一步最重要的工作之一，是继续追加轻雾、中雾、局地团雾、不同光照和不同车流密度下的真实监控视频，形成更具代表性的 \code{benchmark_v2}。

其次，应补齐标准检测指标与更系统的消融实验。当前论文已经用视频级工程指标较清楚地展示了路线优劣，但在更严格的学术表达上，仍应在 UA-DETRAC 验证集上补充 Precision、Recall 和 mAP 等标准检测指标，并对“是否使用深度缓存”“是否使用 \code{patchy} 模式”“是否接入时序过滤 / 二次复核器”等因素进行系统性消融。

再次，应提升 beta 的物理标定能力。当前 beta 已具备连续、稳定和可解释的相对意义，但尚未与路侧能见度仪、气象站或激光雷达观测建立严格映射关系。未来若能够获取这类外部标定数据，则可进一步把 beta 从“相对雾强度代理变量”推进为更接近真实能见度量化的业务指标。

此外，在静止误检治理方面，还需继续扩大 bootstrap negatives 与人工复核规模。当前二次复核器已经证明了训练、加载、打分与回接流程可行，但若要在更复杂场景中具备稳定泛化能力，就必须引入更多真实误检 patch、多视频来源和更细致的人工标签。

最后，在部署层面，本文已经预留了 QAT、INT8、ONNX、TensorRT 与 Jetson 路径。未来可以围绕边缘设备上的推理速度、功耗、时延和稳定性开展更系统测试，使该系统真正从“桌面端研究原型”走向“路侧可部署的小型智能模块”。

若进一步结合陕西省交通运输行业现状，本文项目未来还可以沿三条更具体的应用路径推进：第一，作为既有监测预警系统的视频雾风险识别插件，优先接入监控条件成熟、交通量大的重点线路；第二，围绕西安绕城、新机场线、连霍高速关中段与陕南山区桥隧通道，构建“平原高流量场景”和“山地高风险场景”两类样板验证点；第三，进一步把模型输出从单纯的视频可视化结果提升为可直接进入限速诱导、情报板发布和“一路多方”会商研判的结构化信息\cite{shaanxiMeeting2026,shaanxiWeekly2025,shaanxiRiskProject2025}。

总的来说，本文已经完成了一套从理论方法到工程系统的完整探索。虽然统一模型在车辆检测能力上仍未超越成熟独立检测器，且真实雾视频覆盖、标准检测指标和二次复核器规模仍有待进一步加强，但其方法框架、代码实现、正式基准评估体系和误检治理闭环已经具备较强的继续扩展价值。对于毕业设计终稿而言，当前这版论文与代码、实验产物之间已经能够形成较稳定的一致关系；对于后续继续研究或面向实际部门做小规模原型验证而言，本文也已经具备可复用的工程基础。
